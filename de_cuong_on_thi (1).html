Lab 1 – Routing & Firewall (UFW)
10.0.0.1 | WAN: ens33 | LAN: ens34
▶
ℹ️Nền tảng cho nhiều lab khác. UFW (Uncomplicated Firewall) đơn giản hơn iptables nhiều lần. Làm đúng lab này thì các lab sau sẽ nhanh hơn.
bash
# 1. Bật IP Forwarding (cần cho NAT)
echo 'net.ipv4.ip_forward=1' | sudo tee -a /etc/sysctl.conf
sudo sysctl -p

# 2. Cài đặt và khởi động UFW
sudo apt install -y ufw
sudo ufw --force enable

# 3. Cho phép SSH từ LAN (TRƯỚC khi bật firewall!)
sudo ufw allow from 10.0.0.0/24 to any port 22

# 4. Cho phép forward từ LAN ra WAN (NAT)
sudo ufw default allow routed in
sudo ufw default allow routed out

# 5. Cấu hình NAT trong UFW (thêm vào /etc/ufw/before.rules)
sudo sed -i '/^# Don'\''t delete these required lines/a \*nat\n:POSTROUTING ACCEPT [0:0]\n-A POSTROUTING -s 10.0.0.0\/24 -o ens33 -j MASQUERADE\nCOMMIT' /etc/ufw/before.rules

# 6. Reload UFW để áp dụng NAT
sudo ufw reload
✅Kiểm tra: sudo ufw status numbered | cat /etc/ufw/before.rules | grep -A 3 "\*nat"
⚡Lưu ý: Nếu lần sau muốn xóa toàn bộ rules, dùng sudo ufw reset

Lab 2 – DHCP Server (isc-dhcp-server)
10.0.0.1 | Cấp: 10.0.0.100-200
▶
bash
sudo apt install -y isc-dhcp-server

# Ghi config một lần bằng tee:
sudo tee /etc/dhcp/dhcpd.conf << 'EOF'
default-lease-time 600;
max-lease-time 7200;
subnet 10.0.0.0 netmask 255.255.255.0 {
  range 10.0.0.100 10.0.0.200;
  option routers 10.0.0.1;
  option domain-name-servers 10.0.0.5, 8.8.8.8;
  option broadcast-address 10.0.0.255;
}
EOF

echo 'INTERFACESv4="ens33"' | sudo tee /etc/default/isc-dhcp-server
sudo systemctl restart isc-dhcp-server
sudo systemctl enable isc-dhcp-server
✅Kiểm tra IP đã cấp: sudo cat /var/lib/dhcp/dhcpd.leases

Lab 4 – FTP Server (vsftpd)
10.0.0.3 | Port 21 | Passive: 40000-50000
▶
⚡Phải có allow_writeable_chroot=YES khi dùng chroot_local_user=YES – thiếu → lỗi 500!
bash
sudo apt install -y vsftpd

sudo tee /etc/vsftpd.conf << 'EOF'
listen=YES
anonymous_enable=NO
local_enable=YES
write_enable=YES
chroot_local_user=YES
allow_writeable_chroot=YES
pasv_enable=YES
pasv_min_port=40000
pasv_max_port=50000
EOF

sudo adduser ftpuser
sudo mkdir -p /home/ftpuser/upload
sudo chown ftpuser:ftpuser /home/ftpuser/upload
sudo systemctl restart vsftpd && sudo systemctl enable vsftpd

Lab 6 – DNS Server (Bind9)
10.0.0.5 | Port 53 UDP+TCP | lab.local
▶
bash
sudo apt install -y bind9 bind9utils dnsutils

sudo tee /etc/bind/named.conf.options << 'EOF'
acl trusted { 10.0.0.0/24; localhost; };
options {
  directory "/var/cache/bind";
  recursion yes;
  allow-query { trusted; };
  forwarders { 8.8.8.8; };
  dnssec-validation auto;
};
EOF

sudo tee /etc/bind/named.conf.local << 'EOF'
zone "lab.local" { type master; file "/etc/bind/db.lab.local"; };
zone "0.0.10.in-addr.arpa" { type master; file "/etc/bind/db.10.0.0"; };
EOF

# Tạo zone files từ template rồi chỉnh sửa:
sudo cp /etc/bind/db.local /etc/bind/db.lab.local
sudo cp /etc/bind/db.127 /etc/bind/db.10.0.0

# Kiểm tra cú pháp TRƯỚC KHI restart (quan trọng!):
sudo named-checkconf
sudo named-checkzone lab.local /etc/bind/db.lab.local
sudo systemctl restart bind9 && sudo systemctl enable bind9
✅Test: dig @10.0.0.5 ns.lab.local | nslookup web.lab.local 10.0.0.5

Lab 10 – VPN Server (OpenVPN)
WAN: 192.168.1.10 | VPN: 10.8.0.0/24
▶
⚡Bước gen-dh mất 2-5 phút. Bắt đầu chạy xong làm việc khác trong lúc chờ!
bash
sudo apt install -y openvpn easy-rsa
sudo cp -r /usr/share/easy-rsa /etc/openvpn/easy-rsa
cd /etc/openvpn/easy-rsa
sudo ./easyrsa init-pki
sudo ./easyrsa build-ca nopass        # CN: OpenVPN-CA
sudo ./easyrsa build-server-full server nopass
sudo ./easyrsa build-client-full client1 nopass
sudo ./easyrsa gen-dh                 # ⏱ Chờ 2-5 phút...
sudo openvpn --genkey --secret ta.key

sudo cp pki/ca.crt pki/issued/server.crt pki/private/server.key pki/dh.pem ta.key /etc/openvpn/
sudo systemctl start openvpn@server && sudo systemctl enable openvpn@server

Checklist trước khi nộp bài
▶
  • Ping giữa server và client thành công (2 chiều)
  • Service đang chạy: systemctl status hoặc Get-Service hiện active/running
  • Service tự khởi động: systemctl enable đã chạy
  • Firewall đã mở đúng port cho từng service
  • Test chức năng từ client (không chỉ từ server)
  • Log không có lỗi đỏ: journalctl -u <service> | tail -20
  • Với DNS: named-checkconf và named-checkzone không báo lỗi
  • Với AD DS: client join domain và login domain account thành công
